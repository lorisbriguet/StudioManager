## 1. Resizable Side Peek

**Current:** Fixed `w-1/2` (50% of content area).

**New:** Draggable left edge to resize freely.

### Implementation
- Add a 4px-wide draggable divider between the project list and the peek panel
- Divider: `cursor-col-resize`, subtle hover highlight (`hover:bg-accent/20`)
- On drag: update peek width via `onPointerDown` → `onPointerMove` → `onPointerUp` on the document
- **Min width:** 300px
- **Max width:** 70% of the content area
- **Persistence:** Save width to `localStorage` key `peekWidth`. On mount, read it. Default to 50%.
- Dragging should be smooth — use `requestAnimationFrame` or direct style updates, not React state on every pixel

### Files
- Modify: `src/pages/ProjectsPage.tsx` — replace `w-1/2` with dynamic width + drag divider
- Modify: `src/pages/CalendarPage.tsx` — same if it has a peek panel

## 2. Billing Address — Main Address as Default

**Current:** Client has address fields (address_line1, address_line2, postal_city). Billing addresses are a separate table. If no billing address exists, the section shows "No billing addresses" even though the client has an address.

**New:**
- The billing addresses section shows the client's main address as a read-only entry labeled "Main address" at the top
- Additional billing addresses appear below
- In invoice/quote forms: if no specific billing address is selected, fall back to the client's main address
- The billing address dropdown shows "Main address" as the first option

### Implementation
- `src/pages/ClientDetailPage.tsx`: In the billing addresses section, render the client's own address fields as a styled read-only card at the top, labeled "Main address". Cannot be edited here (edit via the main client fields above). Additional addresses listed below with add/edit/delete.
- `src/pages/InvoiceFormPage.tsx` + `src/pages/QuoteFormPage.tsx`: In the billing address `<Select>`, add a first option "Main address" (value empty or special sentinel). When selected (or when no address is picked), the PDF uses the client's main address fields.
- The PDF components already fall back to client address when `billingAddress` is null — verify this works.

### Files
- Modify: `src/pages/ClientDetailPage.tsx`
- Modify: `src/pages/InvoiceFormPage.tsx`
- Modify: `src/pages/QuoteFormPage.tsx`

## 3. Contacts UI Rework

**Current:** Contact fields (name, email, phone) remain as editable inputs after saving, creating a boxy form-like appearance.

**New:** Contacts display as clean read-only cards:
- Name in `font-medium`, email and phone as muted text on one line
- No input fields visible in read mode
- Edit via pencil icon (click) or right-click context menu
- Editing opens the fields inline (same inputs, but only when editing)
- Context menu: Edit, Delete

### Implementation
- `src/pages/ClientDetailPage.tsx`: Rework the contacts rendering
- Each contact: read-only card showing name, email, phone as text
- Pencil icon on hover (top-right of card)
- Click pencil or double-click → switch to edit mode (inputs appear)
- Save on blur or Enter, cancel on Escape
- Right-click → context menu with Edit / Delete
- "Add contact" button at the bottom

### Files
- Modify: `src/pages/ClientDetailPage.tsx`

## 5. Email PDF via Share Sheet

**Current:** No way to email a PDF from the app.

**New:** "Email PDF" or "Share PDF" option in the context menu.

### Implementation
- Research Tauri share API availability. If `@tauri-apps/plugin-share` or similar doesn't exist:
- **Fallback: AppleScript via Tauri command**
  - Generate PDF to a temp file
  - Use `invoke("share_pdf", { path, to, subject })` Rust command
  - Rust side: use `std::process::Command` to run `osascript` with AppleScript that creates a new Apple Mail message with the PDF attached and the client email pre-filled
  - Subject: "Invoice F-2026-001" or "Quote D-2026-001"
- Add to context menu alongside "Download PDF"
- Add i18n key: `email_pdf` / `share_pdf`

### Files
- Modify: `src/pages/InvoicesPage.tsx` — context menu item
- Modify: `src/pages/QuotesPage.tsx` — same
- Create or modify: `src-tauri/src/lib.rs` — add `share_pdf` command
