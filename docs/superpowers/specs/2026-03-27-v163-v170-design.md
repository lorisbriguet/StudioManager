## 1. Modular Project Pages Rework

### Changes
- Remove `enableModularProjects` toggle from app store and Settings — always-on
- Delete old non-modular code path in `ProjectDetailContent.tsx`
- Replace up/down move buttons with real drag-and-drop via `@dnd-kit`
- Add block width: `"full" | "half"` (default "full")
- Invoices and Quotes blocks default to `"half"` — render side by side in a 2-column row
- Tasks, Workload, Named Tables default to `"full"` — span 100%
- Notes, Resources can be either

### Type Changes
```typescript
export interface LayoutBlock {
  type: BlockType;
  collapsed?: boolean;
  width?: "full" | "half"; // default "full"
}
```

### Grid Layout
- Full-width blocks: 100% width
- Half-width blocks: sit in a CSS grid row `grid-template-columns: 1fr 1fr`
- When rendering, group consecutive half-width blocks into pairs for the grid row
- If an odd half-width block has no pair, it still renders at 50% width

### Default Layout Update
```typescript
export const STANDARD_LAYOUT: LayoutConfig = [
  { type: "tasks" },
  { type: "notes" },
  { type: "workload" },
  { type: "resources" },
  { type: "named_tables" },
  { type: "invoices", width: "half" },
  { type: "quotes", width: "half" },
];
```

### Drag and Drop
- Use `@dnd-kit` `DndContext` + `SortableContext` (already in project)
- Drag handle: `GripVertical` icon on the left of block header
- Reorder persists to `layout_config` JSON on the project

## 2. Calendar Event Colors by Workload Tag

### How It Works
1. Each task checks its workload data for a designated "calendar color" column
2. Fallback chain: designated column → first select/tag column → accent color
3. Value passed through `getTagColor(value, darkMode)` for consistent color
4. Deadlines always red
5. Invoice due dates and quote expiry events keep their existing yellow/red colors

### Workload Column Config
Add `calendarColor?: boolean` flag to the workload column definition. Only one column per project can be designated. Toggle via calendar icon in the workload column editor.

### Calendar Rendering
In `CalendarPage.tsx` events useMemo:
- Look up task's workload cells → find designated column → get value
- `getTagColor(value, darkMode)` → `backgroundColor` and `textColor` on FullCalendar event

### Files
- Modify: `src/types/workload.ts` (add `calendarColor` to column config)
- Modify: `src/components/workload/WorkloadColumnEditor.tsx` (toggle UI)
- Modify: `src/pages/CalendarPage.tsx` (color lookup in events builder)

## 3. Dashboard Presets

### UI
- Remove reset/refresh button from dashboard header
- Replace with preset dropdown: `[Layout: Preset Name v]`
- Dropdown shows: built-in templates, user presets, divider, "Save as..." button
- Select a preset → replaces current layout
- Editing dashboard → shows "Custom" (unsaved)
- "Save as..." → name dialog → saves current widgets + layout
- User presets: rename, delete from dropdown

### Built-in Templates (seeded, not deletable)
- **Financial**: Revenue chart, Outstanding KPI, Expense breakdown, Cash flow, Recent invoices
- **Project Manager**: Active projects, Overdue tasks, Time this week, Planned vs actual, Upcoming deadlines
- **Time Tracker**: Time this week, Weekly trend, Top time consumers, Billable summary, Project time distribution
- **Minimal**: Revenue KPI, Outstanding KPI, Profit margin KPI, Recent invoices

### Storage
New table:
```sql
CREATE TABLE IF NOT EXISTS dashboard_presets (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  layout_json TEXT NOT NULL,
  is_builtin INTEGER NOT NULL DEFAULT 0,
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

`layout_json` contains `{ widgets: WidgetConfig[], layout: GridLayout[] }` — a snapshot of the dashboard store state.

Built-in presets seeded in `ensureSchema()`.

### Dashboard Store Changes
- Add `activePresetId: number | null` to dashboard store
- Add `setActivePreset(id)`, `saveAsPreset(name)`, `deletePreset(id)`
- When layout changes after loading a preset, `activePresetId` becomes null (shows "Custom")

## 4. Wiki Page

### Data Model
```sql
CREATE TABLE IF NOT EXISTS wiki_folders (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS wiki_articles (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  folder_id INTEGER REFERENCES wiki_folders(id) ON DELETE SET NULL,
  project_id INTEGER DEFAULT NULL REFERENCES projects(id) ON DELETE SET NULL,
  title TEXT NOT NULL DEFAULT 'Untitled',
  content TEXT NOT NULL DEFAULT '',
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS wiki_article_tags (
  article_id INTEGER NOT NULL REFERENCES wiki_articles(id) ON DELETE CASCADE,
  tag TEXT NOT NULL,
  PRIMARY KEY (article_id, tag)
);
```

### Page Layout
- **Sidebar** (left, ~200px): folder tree with collapsible items, article count badges
  - "All" entry at top (shows all articles)
  - Folders: create (+), rename (double-click), delete, drag articles between folders
- **Main area**: article list or article editor
- **Top bar**: search + colored tag filter pills + "New article" button

### Article List View
- Shows articles in selected folder (or all)
- Columns: title, tags (colored pills), linked project name, updated date
- Click opens article in editor
- Light dividers, hover states per design system

### Article Editor (Tiptap)
- **Title**: large editable field at top (text-xl font-semibold, no border)
- **Metadata row**: tag pills (add/remove, colored) + project link dropdown (optional)
- **Content**: Tiptap editor with:
  - Slash commands: `/h1`, `/h2`, `/h3`, `/list`, `/link`, `/divider`
  - Keyboard: Cmd+B bold, Cmd+I italic
  - Extensions: StarterKit, Placeholder, Link
- **Auto-save**: on blur or 2s debounce after typing
- **Storage**: content as HTML string

### Project Integration
- Add `"wiki"` to `BlockType` union in `src/types/project.ts`
- Wiki block in modular project layout shows linked articles as compact list (title + tags)
- Click article title → navigates to wiki page with article open

### Sidebar Navigation
- New section label: **"Knowledge"**
- Contains: Wiki (BookOpen icon) + Resources (Database icon)
- Positioned after Time, before Settings divider

### Seeded User Guide
On first run (wiki_folders empty), seed:
- **Folder**: "User Guide"
- **Articles** (one per feature area):
  - Getting Started
  - Clients & Contacts
  - Projects & Tasks
  - Invoices & Quotes
  - Expenses & Income
  - Calendar & Time Tracking
  - Dashboard & Widgets
  - Resources & Wiki
  - Settings & Customization
  - Keyboard Shortcuts

Each article written in the app's current language (use i18n), covering all features with headings and formatted text. Stored as HTML.
