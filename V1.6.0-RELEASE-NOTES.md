## What's New

### 1. Micro-animations

Subtle, Notion/Linear-style motion across the entire app. Every interaction now feels intentional.

- **Task checkbox**: scale pop (0.8 -> 1.15 -> 1) with checkmark stroke draw
- **Task strikethrough**: left-to-right wipe when marking done
- **Status pills**: background-color crossfade on change
- **Page transitions**: fade-in + translateY(4px -> 0) on route change
- **Toast notifications**: slide-in from right with spring bounce
- **Dashboard widgets**: staggered fade-in (50ms delay between widgets)
- **KPI counters**: count-up animation from 0 to value (600ms)
- **List row hover**: smooth background fade (150ms)
- **Sidebar active indicator**: vertical slide to active item
- **Modal open/close**: opacity + scale (0.97 -> 1)
- **Drag items**: box-shadow increase + slight scale-up

**Respects user preferences**: "Reduce motion" toggle in Settings > Appearance. Also respects `prefers-reduced-motion` OS setting. When enabled, all durations set to 0ms.

**Files**: `src/index.css` (tokens + keyframes), `src/hooks/useAnimateIn.ts`, `src/hooks/useCountUp.ts`, `src/stores/app-store.ts`

### 4. Modular Project Pages (Experimental)

Customizable project page layout. Each project can have a different combination of blocks.

**Available blocks**: Tasks, Workload, Resources, Notes, Named Tables, Invoices, Quotes

**Features**:

- Collapsible blocks with section headers
- Reorder blocks (move up/down)
- Add/remove blocks from a menu
- Per-project layout saved as JSON in `layout_config` column
- Predefined templates: Standard, Creative Brief, Production, Simple

**Feature flag**: Must be enabled in Settings > Sandbox > "Modular Project Pages" checkbox. When off, existing fixed layout is used unchanged.

**Files**: `src/components/ProjectBlockLayout.tsx`, `src/components/ProjectDetailContent.tsx`, `src/types/project.ts`, `src/stores/app-store.ts`

### 5. Generate Projects from Quotes

3-step wizard to convert accepted quotes into projects with tasks.

**Flow**: Accept quote -> "Generate project" button -> Review/edit line items as tasks -> Set project details -> Create

Creates project linked to client, tasks from quote line items, and records `converted_to_project_id` on the quote. Full undo support.

**Files**: `src/components/QuoteToProjectWizard.tsx`, `src/db/queries/quotes.ts`, `src/db/queries/projects.ts`

### 6. Flexible Named Tables

Custom spreadsheet-like tables on any project. Multiple tables per project, each with configurable columns.

**Column types**: text, number, checkbox, select (dropdown), tags (multi-select), date

**Features**:

- Add/remove tables per project
- Add/remove columns with type picker
- Inline editing (double-click for text/number, click for checkbox/select/date)
- Collapsible section with table count
- Delete table with confirmation

**Files**: `src/components/NamedTable.tsx`, `src/types/project-table.ts`, `src/db/queries/projectTables.ts`, `src/db/hooks/useProjectTables.ts`

### 7. Time Tracking (Boosted Workload)

Task-based time tracking merged into the workload table. Timer writes directly to `tracked_minutes` on tasks.

**Features**:

- Start/stop timer per task (play/pause button on task rows)
- Only one timer active at a time
- Running timer shown in sidebar
- Tracked time accumulates (re-starting adds to existing)
- Planned minutes from quote-generated tasks for comparison
- Time Overview page: cross-project summary, grouped by project
- Workload system columns: Duration (tracked), Planned (from quotes)

**Files**: `src/hooks/useTimerActions.ts`, `src/pages/TimeTrackingPage.tsx`, `src/components/workload/WorkloadTable.tsx`

### 8. Batch Operations

Multi-select rows on any list page with bulk actions.

**Selection**: Checkbox column, shift+click for range, "Select all" in header.

**Bulk actions by page**:

| Page      | Actions                               |

### 11. Duplicate Expense Warnings

Warns when creating an expense that looks like a duplicate.

**Detection**: Matches 2 of 3 criteria — same supplier (case-insensitive), same amount, same date (+/- 3 days).

**UI**: Yellow warning banner in expense form with "Save anyway" button. Triggered on amount field blur.

**Files**: `src/db/queries/expenses.ts` (`findDuplicateExpenses`), `src/pages/ExpensesPage.tsx`

---

### 12. Global Rate for Line Items

Checkbox on invoice/quote forms to apply the same hourly rate to all line items at once.

When checked, changing the rate updates all existing items and new items inherit it. Unchecking preserves individual rates. When active, the Rate column is hidden and replaced by a label: "All items at XXX CHF/h".

Line items now include a **Unit** column with a dropdown selector (hours, days, units, flat rate). The unit is stored per line item and rendered conditionally in PDF exports.

**Files**: `src/pages/InvoiceFormPage.tsx`, `src/pages/QuoteFormPage.tsx`, `src/components/shared/LineItemsTable.tsx`

## Database Schema Changes

All changes are additive (no breaking migrations):

| Table / Column                   | Type                         | Purpose                              |
| -------------------------------- | ---------------------------- | ------------------------------------ |
| `saved_filters`                  | New table                    | Saved filter configurations per page |
| `project_tables`                 | New table                    | Named table definitions per project  |
| `project_table_rows`             | New table                    | Row data for named tables            |
| `projects.layout_config`         | `TEXT DEFAULT NULL`          | Modular project block layout (JSON)  |
| `tasks.planned_minutes`          | `INTEGER DEFAULT NULL`       | Planned time from quote conversion   |
| `tasks.tracked_minutes`          | `INTEGER NOT NULL DEFAULT 0` | Accumulated tracked time             |
| `quotes.converted_to_project_id` | `INTEGER DEFAULT NULL`       | Link to generated project            |

All added via `addColumnIfMissing()` / `ensureSchema()` in `src/db/index.ts`.

## Known Limitations

- **Modular Project Pages**: Experimental feature behind a flag. May feel rough around the edges. Block reordering uses move-up button, not drag-and-drop (yet).
- **Time tracking**: Timer state does not survive app restart (stored in memory, not DB). Timer is task-level, not project-level.
- **Saved filters**: Filter state is serialized as JSON — changing page filter structure in future versions may invalidate old saved filters.
- **Presentation mode**: Seed data is static SQL. Adding new features in future versions won't automatically update the demo data.
- **Duplicate detection**: Only applies to expenses. Invoice duplicate detection is a candidate for V1.7.0.
